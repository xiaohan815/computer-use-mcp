# 测试笔记

## 问题描述

用户反馈：
1. 坐标转换可能不准确
2. 需要验证 AI 识别效果

## 测试结果

### 坐标转换测试（✅ 通过）

**测试步骤**：
1. 获取初始光标位置：(1068, 214)
2. 移动到目标位置：[682, 441]
3. 再次获取位置确认：(682, 441)

**验证结果**：
- 坐标转换双向正确 ✅
- API → 逻辑：[682, 441] × 1.265 = [863, 558]
- 逻辑 → API：[863, 558] ÷ 1.265 = [682, 441]
- 目标位置与实际位置完全匹配 ✅

### 截图格式测试

#### JPEG Quality 75（Claude Code 默认值）
- 文件大小：~110-130KB（从 PNG 的 ~550KB 减小了 ~75%）
- AI 识别：❌ 对中文识别不够好
- 结论：不推荐用于中文环境

#### JPEG Quality 95（高质量）
- 文件大小：~250-300KB（从 PNG 减小 ~45%）
- AI 识别：⚠️ 仍然不够好
- 结论：不推荐

#### PNG 格式（当前使用）
- 文件大小：~550KB
- AI 识别：✅✅ 完美识别中文
- 坐标转换：✅ 数学上正确
- 结论：✅ 推荐使用

### 格式对比表

| 格式 | 文件大小 | 压缩率 | 中文识别 | 推荐 |
|------|---------|--------|---------|------|
| JPEG 75 | ~120KB | 78% ↓ | ❌ 模糊 | Claude Code 默认 |
| JPEG 85 | ~180KB | 67% ↓ | ⚠️ 一般 | - |
| JPEG 95 | ~300KB | 45% ↓ | ⚠️ 不够好 | - |
| PNG | ~550KB | 0% | ✅✅ 完美 | ✅ 当前使用 |

## 结论

1. **坐标转换是正确的**：数学计算准确，红十字位置与 AI 坐标完全一致
2. **截图格式选择 PNG**：经过测试，JPEG（即使质量 95）对中文识别不够好
3. **当前设置**：PNG 格式（~550KB），确保 AI 完美识别
4. **建议**：
   - 使用键盘导航代替鼠标点击（更可靠）
   - 保持 PNG 格式以确保识别准确性
   - 坐标转换已经正确，不需要再调整

## 优化总结

### 已完成的优化

1. ✅ **Retina 坐标转换修复**
   - 参考 Claude Code 的 `computeTargetDims` 方法
   - 正确处理：逻辑坐标 → 物理坐标 → API 图像坐标
   - 测试验证：误差 < 1 像素

2. ✅ **鼠标移动延迟**
   - 所有鼠标操作添加 50ms settle time
   - 避免触发悬停状态
   - 提高点击准确性

3. ✅ **应用激活功能**

4. ✅ **截图格式**

   - 确保 AI 完美识别中文

## 备选方案

如果未来需要减小文件大小，可以考虑：

**方案 A：降低屏幕分辨率**
- 使用 720p (1280x720) 而不是 Retina 分辨率
- 截图文件会更小，AI 识别可能更准确
- 需要用户手动调整显示设置

**方案 B：使用键盘导航**
- 优先使用键盘快捷键和方向键
- 减少对视觉识别的依赖
- 更可靠，不受截图质量影响

**方案 C：窗口过滤**
- 实现类似 Claude Code 的窗口过滤功能
- 减少干扰，提高识别准确性
- 实现复杂度高，需要原生模块
